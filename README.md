# AI 面试题库问答助手

> 一个基于**真实面经**的 AI 面试问答助手。用户提问某个公司/岗位的面试问题,助手用 **RAG(检索增强生成)** 技术回答:既能溯源真题,又能给出参考答题思路。
>
> 面向秋招 **AI 产品经理 / AI 产品运营** 岗位求职者。

---

## 一、这个项目解决什么问题

- 面经散落在牛客、小红书等平台,收集困难、真假难辨。
- 直接问通用大模型,它会**编造**看似合理但并不真实的"高频题",不可信、无法溯源。
- 学生真正需要的不只是"知道被问了什么",更是"我该怎么答"，以及真实的面试中会问什么。

**核心价值:** ① 基于真实面经,回答**可信可溯源**(区别于通用大模型的幻觉);② 不仅给真题,还补充 AI 答题思路,真正助力面试准备。

---

## 二、技术方案

| 环节 | 选型 |
|---|---|
| 界面 | Gradio |
| RAG 框架 | LangChain |
| 向量化 Embedding | BAAI/bge-small-zh-v1.5(本地开源、免费、中文效果好) |
| 向量库 | FAISS(轻量、检索快) |
| 生成大模型 | DeepSeek(火山引擎方舟接入点) |

**RAG 核心链路:** 用户提问 → 查询理解(指代消解 + 查询改写 + 闲聊过滤) → 向量检索 + 相似度阈值过滤 → 拼接对话历史 → DeepSeek 分层生成答案(真题溯源 / AI 补充思路)。

---

## 三、项目亮点

- **定制化切分**:针对面经的问答结构按"一个 Q+A"切块,而非通用固定长度切分,从源头保证检索单元完整。
- **分层回答设计**:真题溯源严格依据面经、禁止编造来源;AI 补充思路明确标注区分——兼顾可信与实用。
- **数据驱动调参**:相似度阈值(1.1)由采样分数分布得出,而非拍脑袋。
- **多轮对话记忆**:把指代消解、查询改写、闲聊过滤合并成一次"查询理解"调用,检索用改写句、回答用原问题。
- **感知性能优化**:发送后问题即时上屏,再异步生成答案,缩短心理等待。
- **量化评测体系**(离线开发工具):LLM-as-a-Judge 双模型交叉验证,三维评分,支撑数据驱动迭代。

## 四、如何运行

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置 API Key(重要)

本项目不把密钥写死在代码里。运行前请设置环境变量:

```bash
export ARK_API_KEY="你的火山引擎 API Key"
```

> 部署到 Render 时,在后台的 **Environment** 里添加 `ARK_API_KEY`。

### 3. 放入面经数据

把整理好的面经 `.txt` 文件(命名如 `公司_岗位_轮次.txt`,内部用 `Q1/A1/Q2/A2...` 结构)放进 `面经数据/` 文件夹。

### 4. 启动

```bash
python app.py
```

浏览器打开 `http://localhost:7860` 即可使用。

## 五、数据说明与合规

- 面经仅**手动收集自用户自愿公开分享的内容**,不使用爬虫批量抓取,规避版权与反爬风险。
- 出于合规考虑,本仓库为**私有仓库**,面经原文不对外公开;评测仅在本地进行,数据不外传。
